Objetos em Javascript

Usamos chaves {} para designar um objeto usando notação literal. Seus dados, também chamados de propriedades, são organizados usando pares chave-valor. Uma chave é um nome que aponta para uma posição na memória que armazenar um valor de qualquer tipo. Criamos um par chave-valor escrevendo um identificador para servir como chave, seguido de dois-pontos e então o valor. Separamos cada par chave-valor por vírgulas. Chaves são strings, mas o Javascript permite que escrevamos as chaves sem aspas caso não tenhamos nenhum caractere especial, como os espaços. 

          let carro = {
            'Tipo de Combustivel': 'Gasolina',
            cor: 'prata',
            potencia: 115
          };

Propriedades dos objetos

Existem duas maneiras de acessar as propriedades de um objeto, usando a notação com ponto-final e a notação usando colchetes. Com a notação de ponto-final colocamos o identificador do objeto seguido de ponto do nome da chave da propriedade. Já com a notação de colchetes podemos acessar qualquer propriedade, inclusive aquelas que têm como chave uma string. 

          let naveEspacial = {
            planetaOrigem: 'Terra',
            tripulacao:  5,
            'Missão Ativa': true,
            cor: 'prata',
            'Tipo de Combustível': 'Hidrogênio',
            trajeto: ['Vênus', 'Marte', 'Saturno']
        }
        
        var tamanhoTripulacao = naveEspacial.tripulacao; // notação ponto final
        
        var missaoAndamento = naveEspacial['Missão Ativa']; // notação colchetes

Atribuindo valores as propriedades de objetos

Após definirmos um objeto, temos a possibilidade de atualizar os valores de suas propriedades e também criar novas propriedades, uma vez que um objeto é mutável. Podemos usar tanto a notação ponto-final quanto a notação com colchetes junto com o operador de atribuição =, para adicionar novas propriedades ou atualizar propriedades já existentes. Ou seja, duas situações podem ocorrer:

  1. Se a propriedade já existe no objeto, qualquer valor anteriormente armazenado será substituído pelo novo valor atribuído;
  2. Se não existe uma propriedade com aquele nome, uma nova propriedade será adicionada ao objeto.

Note que mesmo um objeto declarado com const pode ter suas propriedades modificadas. O que não podemos fazer é atribuir um novo objeto a essa variável.

            let naveEspacial = {
              planetaOrigem: 'Terra',
              tripulacao:  5,
              'Missão Ativa': true,
              cor: 'prata',
              'Tipo de Combustível': 'Hidrogênio',
              trajeto: ['Vênus', 'Marte', 'Saturno']
          }
          
          naveEspacial.cor = 'branca'; // como a propriedade 'cor' já existe, somente o valor foi atualizado.
          naveEspacial['Nome do piloto'] = 'Bob'; // como a propriedade 'Nome do piloto' não existe, ela foi criada.

Removendo propriedades de objetos

Para deletar uma propriedade, utilizamos a palavra-chave delete seguida da propriedade que queremos.
delete naveEspacial.planetaOrigem;

Métodos dos objetos

OBS: Uma propriedade é o que um objeto tem, enquanto um método é o que um objeto faz.

            let naveEspacial = {
              tripulacao: 5,
              'Missão Ativa': true,
              decolar: function () {
                  console.log('Decolando...');
              },
              pousar() {
                  console.log('Pousando!');
              }
          }
          
          naveEspacial.decolar(); // Decolando...
          naveEspacial.pousar(); // Pousando!

Objetos aninhados

Em aplicações reais, é comum objetos estarem aninhados uns nos outros. 

            let naveEspacial = {
              passageiros: [{nome: 'cachorro espacial'}, {nome: 'gato espacial'}],
              tripulacao: { //objeto e propriedade
                  capitao: { //objeto e propriedade
                      nome: 'Daniel', //propriedade
                      especialidade: 'Engenharia da Computação', //propriedade
                      'comidas favoritas': ['bolo', 'cuscuz', 'pipoca'], //propriedade
                      cumprimentar() { console.log('Olá pessoal, sou o Cap. Daniel.') }//metodo
                  }
              },
              motor: {
                  modelo: 'RocketDyne F1',
                  ano: 2017,
                  potencia: 4400
              }
          }
          
          naveEspacial.tripulacao.capitao.cumprimentar();
          console.log(naveEspacial.passageiros[0].nome);

Objetos passados como referência

Objetos são passados como referência. Isso quer dizer que quando passamos um objeto como argumento de uma função, o computador interpreta esse parâmetro como um apontamento para o espaço em memória em que aquele objeto está armazenado. O impacto disso é que funções que mudam as propriedades de um objeto realmente alteram o objeto permanentemente.

            let naveEspacial = {
              tripulacao:  5,
              'Missão Ativa': true,
              trajeto: ['Vênus', 'Marte', 'Saturno']
          }
          
          function desativarMissao(obj) {
              obj['Missão Ativa'] = false;
          }
          
          let adicionarJupiterNoTrajeto = (obj) => {
              obj.trajeto.push('Júpiter');
          }
          
          desativarMissao(naveEspacial);
          adicionarJupiterNoTrajeto(naveEspacial);
          console.log(naveEspacial); // { tripulacao: 5, 'Missão Ativa': false, trajeto: [ 'Vênus', 'Marte', 'Saturno', 'Júpiter' ] }

Iterando através das propriedades de um objeto

Os pares chave-valor de um objeto não são ordenados, por isso temos uma estrutura chamada for..in que permite que executemos um trecho de código para cada uma das propriedades de um objeto. 

            let naveEspacial = {
              tripulacao: {
                  capitao: {
                      nome: 'Daniel',
                      especialidade: 'Engenharia da Computação',
                      cumprimentar() { console.log('Olá pessoal, sou o Cap. Daniel.') }
                  },
                  medico: {
                      nome: 'Alice',
                      especialidade: 'Medicina',
                      anunciar() { console.log('Realizando exames periódicos na tripulação!') }
                  },
                  cientista: {
                      nome: 'Carla',
                      especialidade: 'Física',
                      pesquisar() { console.log('Iniciando pesquisa sobre a teoria da relatividade...') }
                  }
              }
          }
          
          for (let membro in naveEspacial.tripulacao) {
              console.log(`${membro}: ${naveEspacial.tripulacao[membro].nome}`);
          }
          // capitao: Daniel
          // medico: Alice
          // cientista: Carla

A palavra-chave this

Dentro do escopo de método em objetos, podemos usar a palavra-chave this para fazer referência ao próprio objeto. Dessa maneira, podemos fazer associações e operações usando as próprias propriedades do objeto no qual o método foi chamado.

            const robo = {
              modelo: 'T-800',
              nivelEnergia: 90,
              pegarInfo() {
                  return `Eu sou o ${this.modelo} e meu atual nível de energia é ${this.nivelEnergia}%.`;
              },
              carregarBateria() {
                  this.nivelEnergia = 100;
              }
          };
          
          console.log(robo.pegarInfo()); // Eu sou o T-800 e meu atual nível de energia é 90%.

Construtores

A sintaxe regular de criar objetos usando a notação literal permite criar um único objeto com aquelas propriedades e métodos. Mas existem cenários em que precisamos criar vários objetos similares, como, por exemplo, múltiplos usuários ou itens de menu. Isso pode ser atendido usando as funções construtoras e a palavra-chave new.

Construtores tecnicamente são como funções regulares, que você já aprendeu a criar. Porém, existem duas convenções para criação de construtores:

  1. São nomeados com a primeira letra em maiúsculo.
  2. Eles devem ser executados junto com o operador new.

            function FabricaDeRobos(modelo, material) {
              this.modelo = modelo;
              this.material = material;
              this.nivelEnergia = 100;
              this.pegarInfo = () => `Eu sou o ${this.modelo} e meu atual nível de energia é ${this.nivelEnergia}%.`;
          }
          
          let robo1 = new FabricaDeRobos('R2D2', 'Liga de Titânio');
          let robo2 = new FabricaDeRobos('C3PO', 'Latão');
          
          console.log(robo1.pegarInfo()); // Eu sou o R2D2 e meu atual nível de energia é 100%.
          console.log(robo2.pegarInfo()); // Eu sou o C3PO e meu atual nível de energia é 100%.

Métodos nativos dos objetos: keys(), entries(), assign()

O método keys() (Object.keys(robo);) retorna um array contendo as chaves do objeto passado como argumento.

Dica: Temos também disponível o método Object.values(), que retorna um array contendo os valores do objeto passado como argumento

O método entries() (Object.entries(robo) retorna um array de pares [chave, valor] do objeto passado como argumento.

O método assign() (Object.assign({'Reconhecimento de Voz': true, versao: '1.3.1'}, robo);)retorna um novo objeto contendo a cópia todas as propriedades de um ou mais objetos passados como argumentos.

            const robo = {
              modelo: 'T-800',
              nivelEnergia: 85,
              ativo: true
          };
          
          const chavesRobo = Object.keys(robo);
          console.log(chavesRobo); // [ 'modelo', 'nivelEnergia', 'ativo' ]
          
          const valoresRobo = Object.values(robo);
          console.log(valoresRobo); // [ 'T-800', 85, true ]
          
          const entradasRobo = Object.entries(robo)
          console.log(entradasRobo); // [ [ 'modelo', 'T-800' ], [ 'nivelEnergia', 85 ], [ 'ativo', true ] ]
          
          const novoRobo = Object.assign({'Reconhecimento de Voz': true, versao: '1.3.1'}, robo);
          console.log(novoRobo); // { 'Reconhecimento de Voz': true, versao: '1.3.1', modelo: 'T-800', nivelEnergia: 85, ativo: true }
